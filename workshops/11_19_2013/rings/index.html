What Is It?

The rings visualization is a close cousin to the network, they both require an auxiliary file of nodes and edges. The difference here is that rings doesn't show the whole network but instead an "egocentric" view of a single node's primary and secondary connections. The user is able to then click on a different node to make that the center of attention and navigate through the network one node at a time.

Similar to the network, rings requires setting the .nodes() and .edges() functions which each take arrays in the format specified on the network page. A unique attribute of the rings function is .highlight() method which expects a value of the type specified by the .id() attribute. Thus in the example below we pass an "artist_id" to .highlight(), centering the visualization on that artist.
